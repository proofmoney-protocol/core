# ProofMoney Local MVP Release History v1.0.0

## Purpose

This document summarizes ProofMoney Core release history from v0.1.0-alpha through v1.0.0-local-mvp-freeze.

---

## v0.1.0-alpha

Initial Rust Core MVP.

Established the first local ProofMoney Core structure and early proof model foundation.

---

## v0.2.0-local-ledger

Added local ledger persistence and local state handling.

Established a basic local ledger file model.

---

## v0.3.0-ownership-and-flow

Added ownership and flow concepts.

Introduced local wallet testing, address handling, message signing, transfer event creation, and Proof of Flow.

---

## v0.4.0-proof-explorer-api

Added proof export and local Proof Explorer support.

Established local proof JSON outputs and static local explorer preparation.

---

## v0.5.0-developer-release

Prepared the project for developer review.

Added quickstart docs, demo script, sample fixtures, architecture overview, security review scope, and contributor guide.

---

## v0.6.0-external-review

Prepared external review materials.

Added MVP scope freeze, review index, security review notes, and public developer announcement draft.

---

## v0.7.0-internal-review-hardening

Completed founder-led internal review and hardening.

Added internal review notes, internal findings, documentation safety wording review, and negative test coverage.

---

## v0.8.0-cli-integration-hardening

Hardened CLI integration.

Added isolated CLI integration tests, local ledger reset, local state validation, tamper detection, proof export freshness metadata, and local transfer demo.

---

## v0.9.0-protocol-spec-freeze

Froze current local MVP protocol semantics.

Added specifications for amount model, event schema, proof result schema, release curve, local ledger state, proof export JSON, and wallet ownership safety.

---

## v1.0.0-local-mvp-freeze

First stable local MVP baseline.

Consolidates:

- documentation;
- command reference;
- demo guide;
- risk and limitation summary;
- release history;
- protocol specs;
- safety boundaries;
- baseline status.

---

## Safety Boundary

This release history does not imply mainnet, public network, token launch, sale, airdrop, yield, exchange listing, or audit completion.
